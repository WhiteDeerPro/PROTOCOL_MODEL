# Protocol Model 0.3.0

0.3.0 是一次架构与公开入口重组，也是当前建议用于 GitHub technical preview 的版本。它把开发期的
`0.2.0.dev0` 收敛为可安装、可阅读、可重复生成展示证据的 pre-1.0 快照；这不表示 API 或 requirement
覆盖已经稳定。

## 主要变化

- 建模主线明确为基础语义与 Pattern → `LinkProtocol` → 具体 `VirtualDut` → `SystemProtocol`。
- AMBA link 按 AXI、AHB、APB、ACE、CHI 家族组织；当前可执行重点在 AXI4/AXI4-Lite/
  AXI4-Stream、AHB-Lite/AHB5、APB3/APB4/APB5 和 ACE-Lite ordinary-data profile。
- VirtualDut 通过 typed port 与 attachment 绑定协议；endpoint、address fabric 和有限 AXI→APB
  bridge 由 recipe 装配，不建立按协议名称扩张的设备继承树。
- 运行证据保存 canonical event、fault、resource、provenance、波形源和因果图源；公开 AXI4 入口统一为
  24 个场景，每案都有波形、因果图和机器结果，全套共享一张点到点结构图，其中两案增加展开讲解。
- 新增中英文架构总览与可直接发布的文字/演示材料；推广策略与发布成品分别维护。
- 增加 `pyproject.toml`，Python package 版本由 `protocol_model.__version__` 单点提供。

## 使用边界

- 当前证据来自可执行协议模型和有限场景，不等同于对 RTL 所有执行的证明，也不等同于官方规范逐条覆盖。
- CHI 当前主要提供 link package 与实施边界，ACE-Lite 当前不是完整 coherent execution model。
- AXI→APB bridge 只覆盖文档中声明的同宽、有界 profile；width conversion、更多并发 child 调度和
  通用协议对转译仍在演进。
- 异步 emission、区间时间、网络 wait-for/deadlock 确认和 raw pin/VCD 接入尚未形成完整公共流程。
- 这是 pre-1.0 版本，不兼容 v0.1 API；后续小版本仍可能调整公共入口。

完整变更记录见 [`CHANGELOG.md`](../../CHANGELOG.md)，复现入口见
[`showcase/README.md`](../../showcase/README.md)，长期方向见 [`ROADMAP.md`](../../ROADMAP.md)。

## English summary

Protocol Model 0.3.0 is a pre-1.0 technical-preview snapshot centered on a bottom-up path from semantic
fragments and link protocols to concrete virtual DUTs and system protocols. It adds installable package
metadata, a navigable AXI4 scenario presentation, and bilingual overview material. The generated scenarios
are executable model evidence—not an RTL proof or a clause-by-clause claim of specification completeness.
The compatibility, CHI/coherence, asynchronous execution, network liveness, and raw pin/VCD boundaries
listed above remain active development areas.
