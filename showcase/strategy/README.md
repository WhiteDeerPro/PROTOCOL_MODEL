# 推广策略工作区

本目录保存“怎样传播、向谁传播、凭什么这样表述”的内部工作文档。它帮助维护者选择内容、安排发布节奏并
审计宣称，不是准备原样贴到社区的宣传稿。

## 文档入口

- [对外定位与宣称边界](positioning.md)：目标受众、相邻生态、当前定位和不宜使用的表述；
- [社区推广与采用计划](community-rollout.md)：阶段、渠道、协作角色、衡量指标和调整条件；
- [演示编排策略](demo-program.md)：统一示例入口的阅读层次、讲解密度和后续 bridge 故事；
- [宣称与证据审计](claims-and-evidence.md)：对外文字可以引用的事实、限定条件和升级门槛。

## 与相邻目录的边界

| 位置 | 回答的问题 | 主要读者 | 是否可直接发布 |
|---|---|---|---|
| `showcase/strategy/` | 对谁说、在哪说、按什么阶段说、证据是否足够 | 维护者与 reviewer | 通常不原样发布 |
| `showcase/materials/` | 已选定的内容怎样写成 one-pager、deck、帖子或录屏旁白 | 社区读者 | 是 |
| `showcase/demos/` | 示例怎样声明、运行和重建 | 使用者与贡献者 | 源码和说明可公开 |
| `showcase/generated/` | 本次具名发布实际产生了什么 | 使用者与 reviewer | 由发布脚本生成 |
| `showcase/materials/assets/` | 哪些手工视觉源可嵌入网页或演示稿 | 社区读者 | 是 |
| `docs/architecture/` | 工程对象是什么、边界和机制是什么 | 开发者与研究者 | 是，但不是宣传稿 |

这几类内容可以互相链接，但不互相复制：策略文件不维护逐场景清单；示例导航不讨论渠道和传播指标；对外
材料不自行升级实现状态。场景集合的 canonical 导航位于 [统一 AXI4 示例](../demos/axi4/README.md)，运行证据由
同一集合的具名生成脚本发布。

## 更新顺序

1. 实现或生成证据发生变化时，先更新 [宣称与证据审计](claims-and-evidence.md)；
2. 若目标受众、发布门槛或内容编排需要变化，再更新本目录中的对应策略；
3. 最后修改 [`materials`](../materials/README.md) 中的公开稿和录屏旁白；
4. 示例名称、结果和图片只从 canonical 导航或 generated manifest 引用，不在策略文档重新维护一套数字。

这样可以避免“宣传稿写成现在时 → 反过来迫使实现或文档迎合旧文案”的正反馈。
